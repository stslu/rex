GridViewBase Class

The GridViewBase class - base class of the view used in the grid. It contains a set of properties and methods for control the behavior of the view, which can display data from an abstract Qt model. Detailed Description

The GridViewBase class - base class of the view used in the grid. It contains a set of properties and methods for control the behavior of the view, which can display data from an abstract Qt model. GridViewBase class is the ancestor for all table view - GridTableView and GridBandedTableView.

bool GridViewBase::removeRow(GridRow *row)

Removes the row from the view and model. Before row will be removing, the signal rowRemoving() has raised. In the handler of the rowRemoving() signal you may process some action before. For instance show agree message box or disable the row deletion by set the RowRemovingEventArgs::setHandled() to true of the args.

See also RowRemovingEventArgs and rowRemoving().

bool GridViewBase::removeRows(GridRow *row, int count)

Removes count rows starting with the given row from the view and model. Before rows will be removing, the signal rowRemoving() has raised. In the handler of the rowRemoving() signal you may process some action before. For instance show agree message box or disable the rows deletion by set the RowRemovingEventArgs::setHandled() to true of the args. If the lastRow is NULL then the deletion will be continued till the last row in the view.

See also RowRemovingEventArgs and rowRemoving().

void GridViewBase::setModel(QAbstractItemModel *model, QModelIndex parent = QModelIndex())

Method is intended to connect model with the data to the view. The model should be derived from class QAbstractItemModel. Parameter parent used if the data is not at the first level of the model. createColumns parameter allows automatically create a columns in the view which will correspond to the structure of the model. If this parameter is false then the columns will not be created at all. Later they can be created manually in the required composition. To create a column please use createColumn() function. Note that for banded view, automatic creation of columns works as well. Columns will be created and inserted into the first available band column. If there is no band column in the view it will be created first.

See also model().
